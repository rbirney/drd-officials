# Outer Join

Please enter the following statement which returns all gymmember records:

~~~sql
SELECT * 
FROM gymmember;
~~~

Now, enter the following statement which returns all memberclass records:

~~~sql
SELECT * 
FROM memberclass;
~~~

If you look at the memberclass data, not all of the gym members are taking a class (as yet).

Now, say we want to produce a query which lists all members (name) and the classes (classId) they are taking (if any).
The query would be as follows:

~~~sql
SELECT CONCAT(firstname, " ", lastName) AS Member, classId AS Class
FROM gymmember JOIN memberclass
ON gymmember.memberId = memberclass.memberId
ORDER BY lastName, firstName;
~~~

This will return the same number of records as the previous statement (SELECT * FROM memberclass;)

As you can see a JOIN SELECT will only show the member record if he/she has taken a class. But, how can we return all member records and their classes even if they have NO class record associated? We must use a **LEFT** or **RIGHT** ***OUTER JOIN***. 

An outer join extends the result of a simple join. An outer join returns all rows that satisfy the join condition and also returns some or all of those rows from one table for which no rows from the other satisfy the join condition.
