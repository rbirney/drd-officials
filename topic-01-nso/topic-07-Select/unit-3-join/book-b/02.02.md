# Multi-table Example One

We want to retrieve the list of all classes (class description) that a member takes (firstname and lastname) and
the date they started the class.

There are three things to do here:

1. Identify which tables have the data you are looking for, member name is in the *gymmember* table, class description is in the *fitnessClass* table, and date joined is in the *memberClass* table. The tables required are: *gymmember*, *fitnessClass* and *memberClass*. Since there are 3 tables, we need 2 joins. 
2. Identify which columns are the primary and foreign keys in these tables. There are no foreign keys in *gymmember* or *fitnessClass*.There are 2 foreign keys in *memberClass*. To join the gymmember and memberClass tables we use *memberId*, which is the Primary key in gymmember and a foreign key in memberClass. To join the fitnessClass and memberClass tables we use *classId*, which is the Primary key in fitnessClass and a foreign key in memberClass. We use these in the *ON* clause.
3. Identify which columns we want the query to output; in this case it is *firstName*, *lastName* from the *gymmember* table, *classDescription* from the *fitnessClass* table, and *joinDate* from the *memberClass* table. We will use these in our *SELECT*.

~~~sql
SELECT CONCAT(firstname, " ", lastName) AS Member, classDescription Class, 
       joinDate AS "Start Class Date"
FROM gymmember JOIN memberClass 
ON gymmember.memberId = memberClass.memberId
JOIN fitnessClass 
ON fitnessClass.classId = memberClass.classId
ORDER BY lastname, firstname;
~~~

OR

~~~sql
SELECT CONCAT(firstname, " ", lastName) AS Member, classDescription Class, 
       joinDate AS "Start Class Date"
FROM gymmember JOIN memberClass USING (memberId)
JOIN fitnessClass USING (classId)
ORDER BY lastname, firstname;
~~~
