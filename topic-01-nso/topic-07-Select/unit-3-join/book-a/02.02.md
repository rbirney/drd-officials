# The JOIN statement

Recall the attributes for the Membershiptype and Gymmember tables:

> Membershiptype(memberType, membership, typeDescription, price)

> Gymmember(memberId, firstName, lastName, street, town, county, phoneNumber, emailAddress, dateOfBirth, renewalDate, balance, memberType)

Notice how each of the tables have a common **membertype** column. This column, will be used to JOIN the two tables. It is the primary key in the *Membershiptype* table and the foreign key in the *Gymmember* table.

If we wish to return the name and membership for each gym member, we need values from both tables so we perform the statement as follows:

~~~sql
SELECT CONCAT(firstName, " ", lastName) AS Name, membership As Membership
FROM Gymmember JOIN Membershiptype
ON Gymmember.memberType = Membershiptype.membertype
ORDER BY lastname, firstname;
~~~

![](img/output1.png)

This particular **Join** is also known as an **Inner Join** or **Equijoin**. This is the most common type of **Join** that you will see or use. 

**Note:** The *JOIN* keyword selects all rows from both tables as long as there is a match between the columns. If there are rows in the *Membershiptype* table that do not have matches in the *Gymmember* table, these Books (rows) will NOT be listed.

The following statements are NOT the same. Run both statements and see the different results. Count the number of records returned.

1. Without the **ON** clause:

~~~sql
SELECT CONCAT(firstName, " ", lastName) AS Name, membership As Membership
FROM Gymmember JOIN Membershiptype
ORDER BY lastname, firstname;
~~~


2. With the **ON** clause:

~~~sql
SELECT CONCAT(firstName, " ", lastName) AS Name, membership As Membership
FROM Gymmember JOIN Membershiptype
ON Gymmember.memberType = Membershiptype.membertype
ORDER BY lastname, firstname;
~~~

## Alternative Clauses

The following 2 alternatives will work if the columns that need to be *joined* together have the 
same name:

### USING Clause

~~~sql
SELECT CONCAT(firstName, " ", lastName) AS Name, membership As Membership
FROM Gymmember JOIN Membershiptype USING (memberType)
ORDER BY lastname, firstname;
~~~


### NATURAL JOIN

~~~sql
SELECT CONCAT(firstName, " ", lastName) AS Name, membership As Membership
FROM Gymmember NATURAL JOIN Membershiptype
ORDER BY lastname, firstname;
~~~

**Note:** *NATURAL JOIN* joins all same-named columns (memberType, memberId, firstName, lastName) whether you intend it or not so if the tables you are joining have columns with the same names other than primary key/foreign key fields - it will try to join them.
