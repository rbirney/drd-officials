# Aggregate Functions

SQL provides aggregate functions to help with the summarisation of large volumes of data. For example, we may wish to return the average book price for all books which fall into the *Computing* category. The functions return one **single** value for an entire group or table. The functions operate on sets of rows and return results based on groups of rows.

The common aggregate functions used are:

| Aggregate Function | Description |
| --- | ----------- |
| MIN | returns the smallest value in a given column |
| MAX | returns the largest value in a given column |
| SUM | returns the sum of the numeric values in a given column |
| AVG | returns the average value of a given column |
| COUNT | returns the total number of values in a given column |
| COUNT(DISTINCT) | returns the total number of unique values in a given column |


## Examples

Return the number of rows/records/tuples in the Gymmember table:

~~~sql
SELECT COUNT(memberId) AS "Number of Members"
FROM Gymmember;
~~~

Return the number of pieces of Equipment:

~~~sql
SELECT COUNT(serialNumber) AS "Number of Equipment items"
FROM Equipment;
~~~

Return the number of payments made by Member 107:

~~~sql
SELECT COUNT(paymentId) AS "Number of Payments by Member 107"
FROM Payment
WHERE memberId = 107;
~~~

Return the total payment made by Member 107:

~~~sql
SELECT SUM(amount) AS "Total Payments by Member 107"
FROM payment
WHERE memberId = 107;
~~~

Return the number of members that have made payments:

~~~sql
SELECT COUNT(DISTINCT(memberId)) AS "Number of Members That Made Payments"
FROM Payment;
~~~

Return the average salary of the Trainers:

~~~sql
SELECT AVG(salary) AS 'Average Salary' 
FROM Trainer;
~~~

**Note:** If the Average Salary looks to have a number of decimal places, we will look at rounding in this lab also.
